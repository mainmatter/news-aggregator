# Plan: Auth Routing with Remote Functions

## Overview

Move the news page to `/news` (authenticated), replace `/` with a login page using remote functions, and add auth guards. All auth operations use remote functions (no form actions). Session resolution moves from hooks into a `get_user` remote query.

## Route Structure

```
/       → Login page (email+password + Google), redirects → /news if authenticated
/news   → EditorialDesign (news page), redirects → / if not authenticated
/demo/* → Unchanged
```

## Files to Create

### 1. `src/lib/auth.remote.ts`

Remote functions for all auth operations:

- **`get_user`** — a `query()` that takes an optional boolean argument (default: `false`).
  - Calls `auth.api.getSession({ headers })` via `getRequestEvent()` to resolve the session on demand (no longer done in hooks).
  - If `true` is passed (login page): user exists → `redirect(302, '/news')`, no user → returns `null`.
  - If `false` (default, for protected pages / future remote functions): no user → `redirect(302, '/')`, user exists → returns the user.
  - Usage: `/` calls `await get_user(true)`, `/news` calls `await get_user()`, future remote functions call `await get_user()` to get the authenticated user with automatic redirect.

- **`login_or_register`** — a `form()` with valibot schema: `email`, `_password`, `name` (optional), `action` (picklist: `'login' | 'register'`). Two submit buttons via `fields.action.as('submit', value)`. Calls `auth.api.signInEmail` or `auth.api.signUpEmail`. Redirects to `/news` on success, returns `{ error: string }` on failure.

- **`sign_in_google`** — a `form()` (empty schema). Calls `auth.api.signInSocial` with Google provider and `callbackURL: '/news'`. Redirects to OAuth URL.

- **`sign_out`** — a `form()` (empty schema) so it works without JS. Calls `auth.api.signOut` and redirects to `/`.

### 2. `src/routes/news/+page.svelte`

Renders `EditorialDesign`. Calls `await get_user()` (default=false) — redirects to `/` if not authenticated. Title: "Your News -- Editorial".

## Key Technical Notes

- Session resolution is lazy: `get_user` calls `auth.api.getSession` only when invoked, not on every request
- `get_user(true)` on login page: returns null if not logged in, redirects to `/news` if logged in
- `get_user()` (default false) on protected pages: returns user if logged in, redirects to `/` if not
- `_password` prefix: `form()` convention for sensitive fields (won't repopulate on failed validation reload)
- All `form()` functions handle progressive enhancement automatically (work without JS)
- `sign_out` as `form()` (not `command()`) ensures it works without JS
- `getRequestEvent()` used inside remote functions for headers access
- Google OAuth callback URL: `/news`
- Delete the entire `/demo/` route tree (`src/routes/demo/`) — no longer needed now that real auth lives at `/`.
