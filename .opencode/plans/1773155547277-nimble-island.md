# Editorial Design Refactoring Plan

## Goal

Extract the self-contained editorial design system from `EditorialDesign.svelte` into global CSS variables and reusable atomic components, with a cookie-based dark mode that prevents FOUC.

## Phase 1: Foundation

### 1.1 Install valibot, create shared Article schema + type

- `pnpm add valibot`
- **File**: `src/lib/schemas.ts` (new)
- Define a `article_schema` using valibot (`v.object({ id: v.number(), title: v.string(), ... })`)
- Infer the `Article` type via `v.InferOutput<typeof article_schema>`
- Export both `article_schema` and `Article` type

### 1.2 Replace `layout.css` with editorial design tokens

- **File**: `src/routes/layout.css`
- Replace all existing variables with editorial ones (`--bg`, `--fg`, `--accent`, `--muted`, `--paper`, `--rule`, `--card-hover`, `--font-display`, `--font-body`)
- Light mode as `:root` default
- `@media (prefers-color-scheme: dark)` on `:root:not(.light)` for system dark preference
- `:root.dark` for explicit dark mode
- Move `@keyframes fade-up` and `fade-down` here (currently duplicated in 3 files)
- Move paper texture overlay to `body::before`
- Keep box-sizing reset and base body styles

### 1.3 Update `+layout.svelte`

- **File**: `src/routes/+layout.svelte`
- Replace Playfair Display + Source Sans 3 font imports with Fraunces Variable + Outfit Variable

### 1.4 Dark mode SSR (cookie-based)

- **File**: `src/hooks.server.ts` — Add `handle_theme` using `sequence` with existing `handleBetterAuth`. Read `theme` cookie via `event.cookies.get('theme')`, use `transformPageChunk` to inject `class="dark"` or `class="light"` on `<html>` tag. Store in `event.locals.theme`.
- **File**: `src/app.d.ts` — Add `theme: 'dark' | 'light' | null` to `App.Locals`

### 1.5 Remote function for theme

- **File**: `src/lib/theme.remote.ts` (new) — Export `async function get_theme()` that uses `getRequestEvent()` from `$app/server` to read the `theme` cookie and return it. This allows any component to call `get_theme()` directly without needing a layout server load.
- No `+layout.server.ts` needed — components import `get_theme` from `$lib/theme.remote` directly.
